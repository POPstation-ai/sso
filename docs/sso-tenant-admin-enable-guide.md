概要

テナント管理者の役割は、対象テナントで SSO 機能を利用できる状態にすることだけです。 

テナント管理画面から対象テナントの SSO 機能フラグを有効化します 

Entra ID 側の設定や SSO ログインに必要な接続情報の入力は 顧客側の管理者が行います 関連ドキュメント

SSO機能の有効化手順 

1\. テナント管理画面へアクセス 

POPstation にテナント管理者アカウントでログインし、テナント管理画面を開いてください。 2\. 対象テナントの編集画面を開く 

テナント一覧から SSO を有効化したい対象テナントを選択し、編集画面を開きます。 3\. SSO 機能フラグを有効化 

「SSO」項目を 有効（オン）に切り替えて保存してください。 

これで管理者が SSO 設定画面（ /manage/new/sso-settings ）にアクセスし、Entra ID との接続情報 を入力できるようになります。 

有効化後の引き継ぎ 

管理者に以下を伝えてください。 

1\. SSO 機能が有効化された旨 

2\. SSO 設定の作業手順書: Azure Entra ID 設定ガイド（管理者向け） 

Auth0 ドメイン（ https://auth.POPstation.ai/ ）等の接続先 URL は固定値のため、管理者向 けガイド内に直接記載済み。引き継ぎ時に伝達する必要はありません。 

SSO機能の無効化 

テナント管理画面で対象テナントの「SSO」項目を 無効（オフ）に戻すと、SSO 機能フラグが無効化さ れます。 

既存の SSO 設定値（Entra ID 接続情報）はテナント側に保持されたままです SSO ログイン URL からのアクセスは通常ログイン画面にフォールバックします 再度フラグを有効化すれば、保持されている設定値で SSO ログインを再開できます 

よくある質問  
Q. SSO 機能を有効化しただけで SSO ログインが使えるようになりますか 

いいえ、なりません。管理者が Entra ID 設定 \+ SSO 設定画面への接続情報入力 \+ 有効化を完了して初 めて利用可能になります。 

Q. 管理者が誰か分からない 

対象テナントのユーザー管理画面で、ロールが「管理者」のユーザーを確認してください。 Q. 複数テナントで一括有効化したい 

現在はテナント単位の個別操作のみ対応しています。
